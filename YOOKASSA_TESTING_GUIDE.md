# 🛠️ Инструкция по настройке и тестированию оплаты через ЮKassa

Этот документ содержит полное руководство по настройке личного кабинета **ЮKassa** на основе предоставленных вами параметров и секретного ключа, а также пошаговый план проведения тестового платежа.

---

## 1. ⚙️ Настройка Личного кабинета ЮKassa

Для того чтобы ЮKassa могла отправлять уведомления о платежах вашему приложению (например, о завершении платежа или возвратах), вам необходимо настроить **HTTP-уведомления (Webhooks)** в личном кабинете продавца:

1. Войдите в ваш **Личный кабинет ЮKassa**.
2. Перейдите в раздел **Интеграция** -> **HTTP-уведомления** (или **Уведомления**).
3. В поле **URL для уведомлений** введите адрес вашего бэкенд-обработчика:
   ```
   https://choozi.ru/api/payments/callback
   ```
4. В списке событий выберите следующие события (точно по вашему списку):
   *   `payment.succeeded` (Успешный платёж)
   *   `payment.waiting_for_capture` (Поступление платежа, который нужно подтвердить)
   *   `payment.canceled` (Отмена платежа или ошибка оплаты)
   *   `refund.succeeded` (Успешный возврат денег покупателю)
5. Сохраните настройки.

---

## 2. 📝 Настройка переменных окружения `.env` на VPS

Для проведения тестовых платежей используйте предоставленный вами **тестовый секретный ключ**:

Откройте файл `.env` на вашем VPS-сервере:
```bash
nano /food_app/server/.env
```

Вставьте ваши **Тестовые параметры** ЮKassa:
```env
# --- РЕЖИМ ТЕСТИРОВАНИЯ ЮKASSA ---
YOOKASSA_SHOP_ID=1374499
YOOKASSA_SECRET_KEY=test_GNXpw6Il_Ok4wFOcbUdChawMVxKruQgxC1-_dpSG_R8
```

> [!TIP]
> **Переход в боевой режим:**
> Когда вы будете готовы принимать реальные платежи от пользователей, вам просто потребуется заменить значение `YOOKASSA_SECRET_KEY` на ваш **боевой секретный ключ** (начинающийся с `live_...`) и при необходимости актуализировать `YOOKASSA_SHOP_ID`. Код приложения автоматически и бесшовно адаптируется к боевой среде без дополнительных изменений!

После изменения файла `.env` перезапустите Docker-контейнеры на вашем VPS, чтобы применить изменения:
```bash
docker compose -f docker-compose.prod.yml up -d --build
```

## 🧪 Сценарий проведения тестового платежа (Шаг за шагом)

### Шаг 1: Подготовка аккаунта
1. Откройте сайт **[choozi.ru](https://choozi.ru)**.
2. Войдите или зарегистрируйте постоянный аккаунт с помощью Email и пароля. (Для оформления Premium-подписки обязательна регистрация — это гарантирует сохранность покупки).

### Шаг 2: Инициация оплаты
1. Перейдите в раздел **Профиль**.
2. Нажмите на кнопку **«Разблокировать Premium»** или **«Оформить Premium»**.
3. В открывшемся модальном окне выберите любой тариф (1 Месяц или 12 Месяцев) и нажмите **«Оформить Premium»**.
4. Приложение совершит безопасный запрос к вашему API, создаст платежную сессию в ЮKassa с переданными в метаданных `userId` и `tier` и перенаправит вас на защищенную тестовую страницу оплаты ЮKassa.

### Шаг 3: Прохождение тестовой оплаты
1. На странице оплаты ЮKassa вы увидите уведомление о том, что транзакция проходит в **тестовом режиме**.
2. Выберите способ оплаты банковской картой.
3. ЮKassa сама предложит вам использовать тестовые реквизиты карты (или вы можете ввести произвольный тестовый номер карты и любые CVV/дату действия).
4. Подтвердите тестовый платеж.

### Шаг 4: Магия Webhooks на Бэкенде 🔮
1. ЮKassa в фоновом режиме отправит зашифрованный POST-запрос с событием `payment.succeeded` на ваш URL: `https://choozi.ru/api/payments/callback`.
2. Наш бэкенд для обеспечения 100% безопасности сделает **прямой защищенный запрос проверки** в API ЮKassa по ID платежа (`GET https://api.yookassa.ru/v3/payments/{payment_id}` с авторизацией по вашему секретному ключу).
3. Убедившись, что платеж действительно успешен, бэкенд обновит поле `subscription` пользователя на `premium` в базе данных PostgreSQL и вернет ответ `200 OK` серверу ЮKassa.

### Шаг 5: Редирект и праздничные конфетти 🎉
1. После завершения платежа ЮKassa автоматически вернет вас на сайт:
   `https://choozi.ru/profile?yookassa=success`.
2. Фронтенд считает этот параметр, мгновенно обновит состояние пользователя (запросит свежие данные профиля с бэкенда), отобразит яркое окно успешной покупки с динамическим **салютом из конфетти** 🎉 и разблокирует все премиум-рецепты и функции планировщика меню!
